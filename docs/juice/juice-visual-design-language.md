# Juice Visual Design Language

## Purpose

Juice is not only about attribute syntax.

It also needs a recognizable design language for pages and interfaces to feel professional.

This guide explains the visual principles that have emerged from recent template work.

## 1. Neutral Surfaces First

Most professional layouts do not paint every block with a different color.

The default pattern should usually be:

* white or neutral card surfaces
* restrained accent color
* stronger contrast only where role distinction matters

This makes:

* hierarchy clearer
* pages calmer
* accents more meaningful

## 2. One Primary Accent, Not Many

Most pages should have:

* one dominant accent color
* one optional secondary accent

Too many simultaneous accents make the page feel noisy and immature.

## 3. Display Font + Body Font Pairing

A strong Juice page often benefits from:

* one display font for headings
* one body font for reading

This creates clearer hierarchy than using one voice everywhere.

Examples that worked well:

* `oswald` for display
* `roboto` for body

## 4. Surface Contrast Should Indicate Role

Use stronger contrast when the UI role changes.

Examples:

* white cards for standard content
* dark panel for operational alerts or activity logs
* pale alert card for notices

Do not use strong contrast everywhere.

## 5. Retail, Dashboard, and Marketing Need Different Rhythm

Page types should not all use the same composition language.

Retail:

* shelves
* promos
* categories
* services

Dashboard:

* shells
* summaries
* utility panels
* logs/activity

Marketing:

* hero
* proof
* narrative sections
* CTA rhythm

## 6. Realism Matters

Professional templates feel believable when:

* section roles are clear
* product names sound plausible
* buttons feel like real actions
* color hierarchy supports the product type

This is why some later Game Store and cPanel passes felt more professional than the earlier ones.

## Summary

A professional Juice page usually comes from:

* calmer surfaces
* more disciplined accent usage
* clearer type hierarchy
* role-based contrast
* layout patterns that match the page type
